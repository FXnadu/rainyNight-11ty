# 11ty-deepwhitedocs：内容站模板

这是一个基于 Eleventy 的、经过高度自动化改造的内容站模板，旨在让写作者可以完全专注于内容创作，而将技术细节（如元数据管理）交给脚本自动处理。

## ✨ 核心特性

- **自动化元数据**：文章的标题、发布日期、更新日期、分类等核心信息均可由系统自动生成和维护。
- **约定优于配置**：通过简单的文件名约定，实现文章标题和分类的自动设置。
- **极简写作体验**：在写作时，您几乎不需要关心 front-matter，只需专注于文章本身。
- **一键构建**：通过单一的 `build` 命令，即可完成日期更新、网站构建、资源优化等所有流程。

---

## 🚀 黄金工作流

本项目的推荐工作流程被设计为三个清晰的步骤：

1.  **✍️ 写作**：在 `src/content/posts` 目录下创建和编辑您的 `.md` 文章。
2.  **🛠️ 构建**：在终端运行 `npm run build`，所有自动化处理和网站生成将一键完成。
3.  **🚢 发布**：将 `_site` 目录下的所有文件上传到您的服务器或托管平台。

## ✍️ 如何写作

### 1. 文件位置

所有文章都应存放在 `src/content/posts/` 目录下的对应分类文件夹中。例如：

- `src/content/posts/开发执行篇/`
- `src/content/posts/网站示例篇/`

### 2. 文件名约定（重要）

为了实现标题和分类的自动化，所有文章文件名**必须**遵循以下格式：

`文章标题@分类ID.md`

- **文章标题**：`@` 符号之前的部分，将自动成为文章的 `<h1>` 大标题。
- **`@` 符号**：必须存在，是标题和分类 ID 之间的分隔符。
- **分类ID**：`@` 符号之后、`.md` 之前的部分。这是一个简短的标识符（如 `jssx`, `alzs`），用于在代码中关联分类。它通常与文件所在的文件夹有关。

**示例：**

- 文件名：`测试与验收清单：交付前要确认哪些内容@jssx.md`
- 最终标题：`测试与验收清单：交付前要确认哪些内容`
- 最终分类ID：`jssx`

> **为什么需要 `@` 符号？**
> 这个符号是我们与系统之间的一个明确“约定”。它让脚本可以 100% 准确地知道哪部分是标题，哪部分是技术上用于分类的 ID，避免了任何可能的解析错误。

### 3. Front-Matter (元数据)

得益于自动化脚本，您需要手动编写的元数据被降到了最低。对于一篇新文章，您**最多只需写 `description` 和 `slug` 字段**。

**一个最简化的文章示例：**

```markdown
---
description: "这是对这篇文章的一个简短描述，用于在列表页和搜索引擎中显示。"
slug: "how-to-write-a-great-article" # 可选：为文章设置一个简短的英文/拼音链接别名
---

这里是您的文章正文...

```

- **`description` (推荐)**: 文章的简短描述，用于 SEO 和列表页展示。
- **`slug` (可选)**: 为文章设置一个自定义的、简短的链接“别名”。如果设置了此字段，文章的 URL 将会是 `/posts/how-to-write-a-great-article/`。如果留空，URL 将默认使用中文标题（如 `/posts/如何写一篇好文章@jssx/`）。


**系统会自动为您处理什么？**

当您运行 `npm run build` 时，脚本会自动检查您的文章，并补全或更新以下字段：

- `title`: 来自文件名。
- `date`: 来自文件的创建日期（仅在首次添加时写入，之后不变）。
- `updated`: 来自文件的最后修改日期（仅在内容发生实质性变化时更新）。
- `category`: 来自文件所在的文件夹名称。
- `tags`: 自动添加 `posts`。
- `layout` 和 `permalink`: 自动使用预设的文章配置。

## 🛠️ 命令参考

在开始之前，请先安装项目依赖：

```bash
npm install
```

### 开发预览

```bash
npm start
```

此命令会启动一个本地开发服务器，它会监视您的文件改动并自动刷新浏览器。非常适合在写作时实时预览。

### 生产构建

```bash
npm run build
```

这是您在发布前需要运行的**唯一**命令。它会自动执行一系列任务：

1.  **`update-dates`**: 自动更新所有文章的 `date` 和 `updated` 元数据。
2.  **`clean:site`**: 清理旧的输出文件。
3.  **`sync-meta`**: 同步分类信息。
4.  **`eleventy`**: 执行核心的网站构建。
5.  **`optimize-css-safe`**: 优化和压缩 CSS。
6.  **`perf-self-check`**: 生成一份构建性能报告。

最终的网站成品会输出到 `_site` 目录。

## ⚙️ 站点配置

### 分类简介

- **位置**: `src/content/settings/categoryDescriptions.json`
- **用途**: 为每个分类页面提供独立的简介文本。
- **同步**: 如果您新增了文章分类，可以运行 `npm run sync-meta` 来自动将新的分类结构同步到此文件中，然后您只需补充 `description` 即可。

### 站点全局文案

- **位置**: `src/_data/siteConfig.js`
- **用途**: 修改站点的全局信息，例如：
  - `brand`: 站点名称
  - `navigation`: 顶部导航栏
  - `footer`: 页脚信息
  - `meta`: SEO 相关的元数据（如站点 URL、作者、邮箱）
  - `pages`: 各个独立页面的标题和文案
